Cookie-Free Analytics Setup Guide

No cookies · No GDPR banner · 5-minute setup

Cookie-free analytics tracks visitors without setting cookies, fingerprinting browsers, or storing persistent identifiers. This guide walks through the why and the how — including a working implementation you can add to any website in under five minutes.

Why cookie-free?

Traditional analytics tools (Google Analytics, Mixpanel) rely on first-party or third-party cookies to recognize returning visitors. Cookies require a consent banner under GDPR, CCPA, and ePrivacy rules. Many visitors dismiss or block these banners, degrading data quality.

Cookie-free analytics avoids this entirely. Without cookies, there's no consent requirement for the analytics script itself — you simply don't need a banner. You also get better data: no consent-rate drop, no cookie-blocker gaps.

What cookieless tracking collects

A well-implemented cookieless analytics script collects only non-PII signals:

  • Page path — which page was viewed (no query strings with user IDs)
  • Referrer domain — where the visitor came from
  • Country — derived from IP geolocation (IP itself not stored)
  • Browser and device type — from User-Agent (coarse, not fingerprint-grade)
  • Screen width — for mobile/desktop split (no unique resolution tracking)
  • Language — navigator.language preference

Unique visitor counting uses a daily non-reversible hash of these fields per site. Because the hash resets daily and combines multiple coarse signals, it cannot be used to track an individual across days, sessions, or sites.

Setup with Beam (recommended)

Beam is a managed cookie-free analytics service built on Cloudflare's edge network. It handles the collection, storage, and dashboard — you just add one script tag.

1 Create a free account

Sign up at beam-privacy.com/signup. No credit card required. Free tier: 50,000 pageviews/month.

2 Add your site

After signup, add your domain in the dashboard. You'll get a unique data-site-id for your site.

3 Add the script tag

Paste this into your HTML <head>:

<script defer
  src="https://beam-privacy.com/js/beam.js"
  data-site-id="YOUR_SITE_ID">
</script>

Replace YOUR_SITE_ID with the ID from your dashboard.

4 Verify installation

Visit your site and check the dashboard — your first pageview should appear within seconds. You can also use the Stack Scanner to verify the script is loading correctly.

Manual implementation (self-hosted / DIY)

If you prefer to self-host or build your own collection endpoint, the beam.js source is MIT-licensed and under 2KB. The script sends a JSON POST to your endpoint with: 

{
  "site_id": "your-site-id",
  "path": "/blog/my-post",
  "referrer": "https://news.ycombinator.com",
  "screen_width": 1440,
  "language": "en-US",
  "timezone": "America/New_York"
}

Your server extracts country from the CF-IPCountry header (Cloudflare) or from a MaxMind lookup. It never sees or stores raw IP addresses. See the API reference for the full collection endpoint spec.

Privacy compliance notes

  • No cookie consent banner required (no persistent identifiers)
  • GDPR: no personal data collected — no DPA or data processing agreement needed for analytics
  • CCPA: no sale of personal information — no opt-out mechanism required
  • Do Not Track respected: the script checks navigator.doNotTrack and skips collection if set

Note: Always verify compliance requirements with a qualified legal advisor for your specific jurisdiction and use case.
