Privacy Design

Cookie-Free Tracking Architecture · GDPR Compliant

Beam collects meaningful analytics without cookies, fingerprinting, or personal data storage. This page explains exactly how cookie-free tracking works.

The Problem with Cookie-Based Analytics

Traditional analytics (Google Analytics, Mixpanel, etc.) set cookies to track users across sessions and devices. This creates two problems:

  1. Legal requirement — GDPR, CCPA, and ePrivacy Directive require explicit consent before setting tracking cookies in the EU and California.
  2. Accuracy loss — consent banners result in 20–40% opt-out rates, making data unreliable.

Beam solves both by not using cookies at all.

How Beam Counts Unique Visitors Without Cookies

Beam uses a daily session hash instead of persistent identifiers:

hash = SHA-256(date + path + country + browser + screen_width)

This hash is:

  • Computed fresh each day — cannot track users across days
  • Derived from non-PII fields only — no IP, no email, no full user agent
  • Never stored anywhere — only the count is recorded

The result: Beam can count unique visitors per day without storing any data that would require GDPR consent.

What Beam Does NOT Do

  • No cookie setting — not first-party, not third-party
  • No fingerprinting — no canvas, WebGL, font, or audio fingerprinting
  • No cross-site tracking — data stays scoped to your domain
  • No selling data — your analytics are only yours
  • No IP storage — country is derived then IP is discarded

GDPR Compliance

Because Beam collects no personal data, it falls outside GDPR's scope for most use cases:

  • No consent banner required for the tracking script itself
  • No data subject rights apply (no personal data = nothing to delete or export)
  • No DPA required between you and Beam for core analytics

You should still mention Beam in your privacy policy for transparency. See beam-privacy.com/privacy for sample privacy policy language. For a fuller overview of the technical architecture, see beam-privacy.com/how-it-works.

Infrastructure Privacy

Beam runs on Cloudflare's global edge network:

  • Analytics events processed at the edge closest to the visitor
  • Data stored in Cloudflare D1 (SQLite-compatible)
  • No third-party analytics services involved
  • Dashboard served from beam-privacy.com

Open Source Transparency

The tracking script (beam.js) is fully open source under the MIT license. You can audit exactly what data is sent — no obfuscation, no phone-home behavior beyond the analytics endpoint.
